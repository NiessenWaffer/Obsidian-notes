# Vue.js Fundamentals for Beginners

**Prerequisites:** HTML, CSS, Basic JavaScript
**Date:** 2026-07-06

## What is Vue.js?

Vue is a **progressive JavaScript framework** for building user interfaces (UIs). Unlike full-blown frameworks, Vue is designed to be incrementally adoptable — you can drop it into an existing project or build an entire SPA (Single Page Application) from scratch.

---

## Core Concept: The Vue Instance & Reactive Data

Every Vue app starts by creating an application instance and mounting it to a DOM element.

```js
const app = Vue.createApp({
  data() {
    return {
      message: "Hello Vue!"
    }
  }
})

app.mount('#app')
```

Think of `data()` as the **state** of your component. When `data` changes, the DOM updates automatically. You don't touch the DOM manually — Vue handles it.

---

## Template Syntax (Your HTML + Superpowers)

Vue extends HTML with **directives** — special attributes that start with `v-`.

### 1. Text Interpolation `{{ }}`
```html
<p>{{ message }}</p>
```
Outputs the value of `message`. If `message` changes, the paragraph updates.

### 2. Attribute Binding `v-bind` (or `:`)
Dynamically bind HTML attributes:
```html
<img v-bind:src="imageSrc" />
<!-- shorthand -->
<img :src="imageSrc" />
```

### 3. Event Handling `v-on` (or `@`)
Listen to DOM events:
```html
<button v-on:click="count++">Clicked {{ count }} times</button>
<!-- shorthand -->
<button @click="handleClick">Click me</button>
```

## Reactivity: How Vue Tracks Changes

Your data is wrapped in reactive proxies. When you do:
```js
data() {
  return {
    name: "Wilcon"
  }
}
```
Vue tracks every access and mutation. When `name` changes, only the parts of the DOM that use `name` re-render. This is called **fine-grained reactivity**.

---

## Conditional Rendering `v-if` / `v-show`

```html
<p v-if="isLoggedIn">Welcome back!</p>
<p v-else>Please log in</p>

<p v-show="isVisible">This toggles with CSS display</p>
```
- `v-if` — Actually removes/adds the element from the DOM. Great for **one-time toggles** (login/logout).
- `v-show` — Toggles `display: none` CSS only (element stays in DOM). Faster for **frequent toggles** (tabs, accordions).

## v-if vs v-show Summary

| | v-if | v-show |
|---|---|---|
| DOM | Removes/adds element | Always renders, toggles CSS `display` |
| Initial render | Nothing if false | Always renders (even if hidden) |
| Toggle cost | Expensive (creates/destroys) | Cheap (just CSS) |
| When to use | Rare changes, login state | Frequent toggles, tabs |
| `v-else` support | Yes | No |

## Looping with `v-for`

Render lists from arrays:
```html
<ul>
  <li v-for="(item, index) in items" :key="item.id">
    {{ index + 1 }}. {{ item.name }}
  </li>
</ul>
```
**Always use `:key`** for performance — it helps Vue track each element.

---

## Two-Way Binding with `v-model`

The magic of forms: bind an input to data and keep them in sync.
```html
<input v-model="username" />
<p>Your username is: {{ username }}</p>
```
Change the input → `username` updates. Change `username` programmatically → input updates.

---

## Computed Properties

Derived values that cache their result and only re-compute when dependencies change:
```js
computed: {
  fullName() {
    return this.firstName + ' ' + this.lastName
  }
}
```
Use computed over methods when you want **caching** based on reactive dependencies.

---

## Methods

Functions tied to a component:
```js
methods: {
  greet() {
    alert('Hello ' + this.name)
  }
}
```

## How Vue Templates Work (Under the Hood)

Vue templates are **valid HTML** — your browser can read them fine. But Vue does extra work:

1. **You write HTML templates** (like `<p>{{ message }}</p>`)
2. **Vue compiles them into optimized JavaScript** at build time
3. **The reactivity system** tracks data changes
4. **Vue updates only the minimum DOM** needed — no wasted work

This means you write simple HTML with some extra syntax (`{{ }}`, `v-if`, `:bind`), and Vue handles the heavy lifting of DOM updates for you.

### Render Functions (Advanced)

Instead of templates, you can write pure JavaScript:

```js
// Template way (easier)
<h1>{{ title }}</h1>

// Render function way (manual)
render() {
  return h('h1', {}, this.title)
}
```

Templates are recommended because Vue can optimize them at compile time. Render functions give more control but lose some of that optimization.

### The Big Picture

```
Your HTML Template
       │
       ▼
  Vue Compiler  ──► Optimized JS Code
       │
       ▼
  Reactivity System ──► Tracks data changes
       │
       ▼
  Smart DOM Updates (minimum changes only)
```
